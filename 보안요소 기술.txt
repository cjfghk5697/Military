1.인증기술
사용자 인증: 통신하고 있는 상대가 내가 원하는 상대인지 확인할 수 있도록 해주는 기술

Chalenge-Response방식
-사용자가 서버에 암호화 키자체를 직접적으로 전송하지 않으면서
 해당 비밀키나 개인키를 소용하고 있는 정당한 사용자임을 증명
-매번 새로운 세션의 차별성을 위해 타임스탬프/세션 랜덤값/ 순서번호를 이용할 수 있음
-대칭키를 이용한 방법
1)사전에 사용자와 서버간의 대칭키를 공유하는 방식
2)사용자가 서버로 접속해 서버로부터 난수 값을 받아서 그걸 대칭키로 암호화해서 보내는 것
-공개키를 이용한 방법
1)사용자가 서버로 접속해 서버로 부터 난수 값을 받아서 그걸 개인키로 암호화해서 보내는 것

2.메시지 출처 인증 기술
-메시지를 암호화 하는 방법
1)대칭키를 이용해 암호화
2) 비대칭키를 이용한 암호화

-메시지 인증코드(MAC)
메시지와 대칭키를 입력으로해 인증값을 쓰기위해 만들어진 코드
사전에 송신자와 수신자 간의 대칭키의 공유가 필요

-해시함수
데이터를 정해진 크기의 Message Digest로 만드는 일방향 함수
MAC과 달리 대칭키를 사용하지 않기에 키교환 X
해시함수의 결과값 자첸느 기밀성이 없어서 암호화와 섞어서 사용

3. 디바이스 인증기술
-ID/PW기반 디바이스 인증
1)일반적으로 사용되는 인증방식
2)SSID방식/WEP인증방식/RFID와 리더기간의 인증방식 등

-MAC주소 기반 디바이스 인증
1)접속하는 단말기의 MAC주소값을 인증서버 또는 AP에 등록해서 인증받는 방식
2)AP에서 사용됨

-암호 프로토콜을 이용한 디바이즈 인증
AP를 통한 정보자산의 불법적인 접근 또는 기나 세션을 품쳐 정보를
-Challenge-Response
OTP에서의 Challenge-Reponse와 유사하게, 일회성 해시값을 생성해 사용자를 인증
인증서버가 난수를 만들어 클라이언트로 전송하면 클라이언트는 패스워드 해쉬 알고리즘을 적용해 반환

4. 접근통제정책
-DAC(임의적 접근통제)
신분기반정책, 어떤 사용자든 임의적으로 객체에 대한 다른 사용자들의 접근을 허용할 수 있는 기법
모든 주체와 객체에 대해 일정. ACL을 활용/ 중앙집중적으로 통제되는 환경에서 부적합
결론적으로, 사용자가 누구고 어떤 그룹에 속해있는지에 따라 접근 통제가 이뤄짐.

-MAC(강제적 접근통제)
규칙기반, 관리자가 주체들에겐 허가등급을 가각의 객체에는 비밀등급을 부여해서 허가등급과 객체의 비밀등급을
따져 접글을 제어. 최상이어도 모든 객체 열람은 불가능.
통제가 용이하고 보안관리자 주도하에 중앙 집중적 관리가 가능하다는 장점

-RBAC(역할 기반 접근통제)
역할 기반 정책의 적용/ 임의적 접근 통제와 강제적 접근통제의 단점을 보완한 기법
역할들을 생성해서 역할마다 권한을 준다음 사용자들에게 각 역할을 부여함, 즉 사용ㅈ마의 역할에 따라 접근을 제어

-접근통제행렬과 ac
주체를 열에, 객체를 행에 표시해서 각각에 권한을 부여하느 방법
AC(접근통제)

5. 키 분배 프로토콜
1) KDC기반 키분배
두가입자 사이에 공통의 암호키를 소유할 수 있고 키분배 과정을 수행하는 신뢰된 기관
간단히 말해, 비밀키를 만들어서 암호화 통신을 원하는 사람들에게 키를 나누어줌

2)Needham-Schroeder프로토콜,diffie-hellman
과정 참고바람, 재전송 공격에 취약(MITM)

3)RSA이용키 분배방법
소인수 분해의 어려움을 이용하는 암호 알고리즘

6.전자서명
사용자 공개키와 사용자 id정보를 결합해 인증기관이 서명한 문서
-PKI
공개키 인증서의 인증성을 제공하기 위한 신뢰구조
-CRL
인증서 취소목록
-OCSP
실시간으로 인증서가 유효한지 검증하는 프로토콜
